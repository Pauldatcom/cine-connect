# Bug reports

Open a **GitHub Issue** (or your team’s tracker) with enough context to reproduce the bug. Vague reports waste time.

## Title

Short and specific: `fix: watchlist 500 when film deleted from DB` — not `bug` or `it broke`.

## Template

**Environment**

- OS:
- Node / pnpm version:
- Branch or commit SHA:

**Steps to reproduce**

1. …
2. …

**Expected**

What should happen.

**Actual**

What happened (error message, status code, screenshot if UI).

**Logs / screenshots**

- Backend log snippet, browser console, or network tab (redact tokens and secrets).

**Optional**

- Minimal repro repository or curl command
- Suspected component (e.g. `backend/src/routes/watchlist.ts`)
